Segment Anything | 语义分隔在Stable Difussion中应用测试

发表于2023-08-18|更新于2023-08-21|炼丹
|阅读量:

概括

SDwebui插件，用于分隔图像元素、自动生成蒙版、定向修改图片元素、简化抠图流程等。
项目地址：continue-revolution/sd-webui-segment-anything: Segment Anything for Stable Diffusion WebUI (github.com)

已测试：

Single Image、GroundingDINO

待测试：

Batch Process、Auto SAM、Upload Mask to ControlNet Inpainting

安装方式

分成两步，安装插件本体+下载对应模型。
模型中越大的占用显存越多，从高到低分别为h、l、b。

使用方法

基础单图模式 Single Image

选择合适的模型，上传图片
logseq.order-list-type:: number
因为电脑只有4060，怕8G显存扛不住大模型，所以先选用最小的b模型试试。

左键单击想识别的部分，右键单击不想识别的部分
logseq.order-list-type:: number
每次单击会创建一个点，左键是黑点，右键是红点，再次左键单击可以删除不想要的点。
官方原文：Left click the image to add one positive point (black dot). Right click the image to add one negative point (red dot). Left click the point to remove it.

使用GroundingDINO可以实现通过提示词识别图片元素，但这个东西和SAM的点提示point prompts有冲突，一会再讲到。
预览识别图片
logseq.order-list-type:: number
单击Preview Segmentation
可以看到，SAM自动生成了三个蒙版以及从蒙版中提取出的元素。
选择一个喜欢的效果，不行就重新跑一遍。
注意，选择中第一列对应0，第二列对应1，第三列对应2。

重绘蒙版
logseq.order-list-type:: number
完成以上部分后，我们得到了三张蒙版和初步提取出的png图片。
那有什么用呢？
其一，提取出的png图片可以扔进PS里当素材，相当于简化抠图流程。
其二，蒙版可以扔进蒙版重绘或者ControlNet里面，重绘对应内容。
这就可以应用在修手、换衣服、换脸等方面了。
因为做演示时没想太多，因此这里先不做重绘蒙版的演示了。

语义分割 Single Image + GroundingDINO

选择模型，上传图片
logseq.order-list-type:: number
第一步还是一样，这里我换了张图片，为后续换发色做准备。

配置GroundingDINO语义分割
logseq.order-list-type:: number
首先时选中Enable GroundingDINO。
这里插件会有提示，把它看成是告诉你GroundingDINO和point prompts有冲突就行。
解决冲突的办法就是确保图片上没有点，途径是上传新图片或点一下Remove all point prompts
然后在Detection Prompt里输入我们想要识别的内容，比如说这次我想修改的头发。
最后点一下预览，看看识别结果。
这里第一次使用的小伙伴们得注意了，插件会自动下载相应文件，等待就好。
最后生成了这么个东西：

箱体是我们想识别的元素所在位置，这里因为相关元素分布集中所以影响不大，如果修手的话就会理解箱体到底有什么用了。
我观察到1覆盖不完全，所以选择了0。（箱体序号在左上角）
然后再次预览分割结果就行。
Emm……不知道为啥，效果不理想，AI把整个人都识别出来了。
logseq.order-list-type:: number
猜测原因是SAM模型不太行，或者是GroundingDINO提示词不够具体，只写了hair，没有反向提示词（话说这玩意儿也配置不了反向提示词吧）
先到这里吧，等碰到修手的时候再更新下一段。
